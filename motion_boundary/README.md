# Motion-State BoundaryHead

This folder is a standalone experiment workspace for learning skill-boundary scores from CALVIN
action/state history. It is intentionally separate from the main HiMem-Bridge-VLA model path.

## Core Hypothesis

Low-level motion history contains a stable signal for intent termination:

```text
past action/state window -> boundary score
```

The module does not predict a skill id, does not produce a latent `z_skill`, and does not implement
a planner. It only estimates:

```text
P(current motion intent has terminated | past action/state)
```

## Data

The first supported data source is the LeRobot-style CALVIN conversion already used by this
repository:

```text
datasets/calvin/lerobot/task_D_D/
  meta/tasks.jsonl
  meta/episodes.jsonl
  data/*/*.parquet
```

Boundary supervision comes from the JSONL sidecar generated by:

```bash
python scripts/prepare_calvin_boundaries.py \
  --auto-lang-ann datasets/calvin/original/task_D_D/training/lang_annotations/auto_lang_ann.npy \
  --output datasets/calvin/annotations/task_D_D_boundaries.jsonl
```

Each row stores a language segment:

```json
{"segment_id": 0, "start": 10, "end": 50, "task": "open_drawer", "skill_id": 3, "language": "open drawer"}
```

The segment end is treated as a weak skill-termination event.

## Features

Default input features:

```text
action
robot state
delta action
delta robot state
gripper transition
```

The main model should not use CALVIN `scene_obs`, because that is privileged simulator state. It can
be added later as an oracle-only upper bound.

Feature normalization is disabled by default to avoid per-trajectory future leakage. If normalization
is enabled for offline probing, replace it with training-set statistics before using the head online.

## Labels

For every event frame `e = segment_end`:

```text
positive zone: |t - e| <= positive_radius
ignore zone:   positive_radius < |t - e| <= ignore_radius
negative:      all other valid frames
```

The default uses soft labels:

```text
y_t = exp(-|t - e| / label_sigma)
```

The ignore zone is excluded from the loss. This matters because language segment endpoints are weak
annotations, not exact physical contact-switch timestamps.

## Model

The default model is a multi-scale causal TCN:

```text
[B, W, D]
  -> Linear(D, hidden_dim)
  -> CausalConv dilation 1
  -> CausalConv dilation 2
  -> CausalConv dilation 4
  -> CausalConv dilation 8
  -> last token
  -> MLP
  -> boundary_logit
```

The sigmoid of the logit is the boundary score.

## Training

Run from the repository root:

```bash
python -m motion_boundary.train \
  --config motion_boundary/configs/default.yaml \
  --run-dir motion_boundary/outputs/calvin_tcn_v0
```

On the AutoDL server used for the first CALVIN run:

```bash
/root/autodl-tmp/miniforge3/envs/Evo1/bin/python -m motion_boundary.train \
  --config motion_boundary/configs/autodl_calvin_tcn_v0.yaml
```

The training objective is masked weighted BCE over valid frames:

```text
loss = BCEWithLogits(logit_t, y_t) * valid_mask
```

The default sampler is event-centered:

```text
50% positive windows
25% hard negative windows
25% easy negative windows
```

The default config uses a trajectory-level random split for in-distribution validation. For testing
task generalization, use the task-heldout config:

```bash
/root/autodl-tmp/miniforge3/envs/Evo1/bin/python -m motion_boundary.train \
  --config motion_boundary/configs/autodl_calvin_tcn_task_holdout.yaml
```

## Evaluation

Evaluate a saved checkpoint:

```bash
python -m motion_boundary.evaluate \
  --config motion_boundary/configs/default.yaml \
  --checkpoint motion_boundary/outputs/calvin_tcn_v0/best.pt
```

The evaluator reports frame-level AUPRC and event-level metrics after peak detection:

```text
score >= threshold
score is local maximum
cooldown satisfied
```

Threshold calibration:

```text
tau_p: highest threshold that still reaches planner_recall_target
tau_w: lowest threshold that reaches memory_precision_target
```

The default intent-replan target is recall 0.90. The default memory-write target is precision 0.95.

When evaluating a checkpoint that already stores its training config, `--config` still overrides the
runtime data/evaluation/output sections. This is useful for threshold-only sweeps without retraining.

## Initial Experiment Matrix

Recommended first runs:

```text
H0: heuristic speed/jerk/gripper baseline
B1: TCN action-only
B2: TCN action + robot state
B3: TCN action + robot state + soft labels
O1: TCN action + robot state + scene_obs, oracle upper bound only
```

The main branch should start from B2/B3 before adding planner behavior.
