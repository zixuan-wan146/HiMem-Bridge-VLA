# 2026-06-18 To-Do

## Goal

Implement the first Coarse Planner path end to end, then split it into a standalone warm-up module before relying on joint training:

```text
CoarsePlanner(H_t_vlm, s_t) -> plan_tokens
plan_tokens + memory_tokens -> BridgeAttention condition path
coarse future action targets -> planner supervised loss
```

## Design Constraints

- The first planner does not read memory.
- Plan tokens and memory tokens are parallel BridgeAttention conditions.
- Planner architecture is Transformer-based and uses at least three layers.
- Future demonstration actions are labels only; they are never planner inputs.
- Coarse target generation must account for episode tail padding with `coarse_action_mask`.
- Coarse target cache keys must include planning horizon and plan-step config.
- Standalone planner training must reuse the same `CoarsePlanner` model class used by the main model.
- The dedicated planner dataset should cache `vlm_tokens`, `state`, `coarse_actions`, and `coarse_action_mask`.
