The target path is:

```text
Stage 1:

  z_hat_i = W_z P_tau_i

Inference:
  P_active = PlanTokenQueue(P_tau, executed_steps=N)
  BridgeAttention(H_t_vlm, s_t, P_active, M_t)
  ActionHead(...)
```

## Finalized Design Constraints

- Delete the old compressed coarse-action supervision path.
- Planner still outputs plan tokens `P`, and inference still injects only plan
  tokens into BridgeAttention.
- Action-segment AE is action-only. It does not use state, VLM tokens, memory, or
  language as encoder/decoder conditions.
- AE latent and decoder reconstruction losses shape planner tokens only during
  training.
- Plan cache is consumed by cumulative executed low-level steps `N`, not by number
  of inference calls.
- Token suffix index is:

```text
u = floor(N / c)
r = N mod c
P_active = P_tau[u:K]
```

- Refresh when cache is empty, episode reset happens, transition trigger fires, or
  `N + requested_execute_steps > H_p`.
- Strict training samples use an anchor/current split:

```text
P_tau = f_plan(H_tau, s_tau)
t = tau + u * c
ActionHead(H_t, s_t, P_tau[u:K], M_t)
```

- BridgeAttention must support padded variable-length plan suffixes via a
  plan-token mask.
- Keep network design knobs in config, but do not treat the exact AE architecture
  as final.

planner. The current best v17 planner reaches the mid-term gate and is in the
0.08-ish usable range, but does not strictly cross `0.08`.

Working target:

```text
near-term gate: raw_latent_mse <= 0.12
mid-term gate:  raw_latent_mse <= 0.10
usable target:  raw_latent_mse ~= 0.08
```

standalone planner into the larger model. It should be judged together with:

```text

token/suffix error: no severe late-token or u in {0,2,4,6} collapse
```
